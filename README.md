## 소개

**3D Mini Golf**는 물리 엔진을 활용한 브라우저 기반 미니골프 게임입니다. 별도의 설치 없이 웹 브라우저만으로 즐길 수 있으며, 모바일과 데스크탑 환경을 모두 지원합니다.

- 총 **9홀** — 직선 코스부터 회전 장애물, 바운스 범퍼, 경사 구역, 언덕까지 다양한 기믹 제공
- **물리 기반 홀컵** — 공이 실제로 홀컵 구멍에 빠져야 홀인 처리
- **솔로 모드** — 원하는 홀부터 시작 가능한 스테이지 선택

---

## 플레이 방법

1. 메인 메뉴에서 **솔로 플레이** 선택
2. 시작할 홀 번호 선택 (1~9홀)
3. 드래그 또는 키보드로 방향과 파워를 조절하여 샷
4. 공을 홀컵에 넣으면 다음 홀로 자동 진행
5. 전체 홀 클리어 후 최종 스코어카드 확인

## 조작법

### 모바일 (터치)

| 조작 | 설명 |
|------|------|
| 한 손가락 드래그 | 방향 및 파워 설정 (슬링샷 방식) |
| 손가락 떼기 | 샷 발사 |
| 두 손가락 드래그 | 카메라 회전 |
| 두 손가락 핀치 | 카메라 줌 인/아웃 |

### 데스크탑 (마우스 + 키보드)

| 조작 | 설명 |
|------|------|
| 좌클릭 드래그 | 방향 및 파워 설정 |
| 마우스 떼기 | 샷 발사 |
| 우클릭 드래그 | 카메라 회전 |
| 스크롤 휠 | 카메라 줌 인/아웃 |
| `A` / `D` | 조준 방향 회전 |
| `Space` 홀드 | 파워 충전 |
| `Space` 떼기 | 샷 발사 |

## 코스 구성

| 홀 | 파 | 기믹 | 설명 |
|----|-----|------|------|
| 1 | 2 | 언덕 | 중앙 언덕을 피해 직선 공략 |
| 2 | 3 | 경사 + 벽 범퍼 | 경사 구역이 공을 왼쪽으로 밀어냄 |
| 3 | 2 | 언덕 | 좁은 통로, 오프센터 언덕으로 정밀 조준 요구 |
| 4 | 3 | 경사 | 오른쪽으로 밀리는 경사, 보정 조준 필요 |
| 5 | 4 | 회전 바 + 언덕 | 회전 장애물 타이밍 맞추기 |
| 6 | 3 | 바운스 범퍼 + 언덕 | 핀볼 스타일 3개 범퍼 |
| 7 | 4 | 이중 회전 바 + 언덕 | 반대 방향으로 도는 두 개의 바 |
| 8 | 4 | S자 게이트 + 언덕 | 지그재그 통로와 원거리 경사 |
| 9 | 3 | 회전 바 + 범퍼 + 언덕 | 고속 바와 좌우 범퍼로 둘러싸인 홀컵 |

**OB(Out of Bounds)**: 공이 코스 밖으로 나가면 +1 벌타 후 마지막 정지 위치에서 재시작

## 프로젝트 구조

```
src/
├── main.ts          # 씬, 물리 월드, 게임 루프, 상태 머신
├── MainMenu.ts      # 메인 메뉴 (솔로/멀티 선택, 스테이지 선택)
├── Course.ts        # 9홀 코스 정의, 물리/시각 요소 생성
├── Ball.ts          # 골프공 물리 바디 + 메시, 홀인 애니메이션
├── InputManager.ts  # 터치/마우스/키보드 통합 입력 처리
├── ShotArrow.ts     # 조준 화살표 + 탄도 점선 미리보기
└── ScoreCard.ts     # HUD, 홀 완료 메시지, 최종 스코어카드
```

## 로컬 실행

**요구 사항**: Node.js 18 이상

```bash
# 저장소 클론
git clone https://github.com/Hwanje/golf-game.git
cd golf-game

# 의존성 설치
npm install

# 개발 서버 실행 (localhost:5173)
npm run dev

# 프로덕션 빌드
npm run build

# 빌드 결과물 미리보기
npm run preview
```

## 주요 구현 사항

### 물리 기반 홀컵
지면을 4개의 `CANNON.Box`로 분할하여 홀 위치에 실제 물리적 구멍을 생성합니다. 공이 구멍 위를 지나면 중력에 의해 실제로 빠져 들어가며, `y < -0.12` 조건으로 홀인을 판정합니다.

### 회전 장애물
`CANNON.Body.KINEMATIC` + `angularVelocity`를 설정하여 물리 엔진이 쿼터니언을 직접 적분합니다. 공과의 충돌 시 자연스러운 물리 반응을 제공합니다.

### 경사 구역 (Slope Zone)
별도의 기울어진 지형 대신, 공이 지정 직사각형 영역 내에 있을 때 매 프레임 `applyForce()`로 힘을 가하는 방식으로 구현되어 있습니다. 성능과 안정성이 우수합니다.

### 언덕 (Hill)
`CANNON.Sphere`를 지면 아래 깊이 묻어 상단 돔만 지면 위에 노출시키는 방식입니다. 공이 실제로 완만한 언덕을 타고 넘는 물리 반응을 구현합니다.

### 바운스 범퍼
별도의 `bouncyMaterial` (restitution 0.88)을 적용한 실린더로, 공이 맞으면 강하게 튕겨나갑니다.

### 상태 머신
```
MENU → WAITING → SHOOTING → HOLE_IN → (다음 홀 or GAME_OVER)
                    ↑ OB 발생 시 WAITING으로 복귀
```
